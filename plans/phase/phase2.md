# Phase 2 — 프로젝트 & 컬렉션 & 환경변수

## 참고 Phase

| 구분 | Phase |
|------|-------|
| 선행 | Phase 1 — 요청 편집기·히스토리 기반 위에 저장 기능 추가 |

---

## 목표

요청을 프로젝트 단위로 묶어 저장·재사용하고, 환경별 변수를 분리해 관리.

---

## 데이터 계층

```
프로젝트 (Project)
  └─ 컬렉션 (Collection)
       └─ 요청 (Request)
```

---

## 체크리스트

### 프로젝트

- [ ] 프로젝트 생성 / 이름 변경 / 삭제
- [ ] 프로젝트 전환 (활성 프로젝트 기준으로 사이드바 표시)
- [ ] 프로젝트별 환경변수 분리

### 컬렉션

- [ ] 컬렉션 생성 / 이름 변경 / 삭제
- [ ] 프로젝트 내 컬렉션 목록 사이드바 TreeView 표시

### 요청 저장

- [ ] 현재 편집 중인 요청을 컬렉션에 저장
- [ ] 요청 이름 변경 / 삭제 / 순서 변경 / 복제
- [ ] 클릭 시 요청 편집기에 복원

### 검색

- [ ] 컬렉션·히스토리에서 요청 이름·URL 기준 검색

### 환경변수

- [ ] 프로젝트 내 환경(Environment) 생성·전환 (예: dev, staging, prod)
- [ ] `{{변수명}}` 형태로 URL·Headers·Body에서 치환
- [ ] 변수 편집 UI

### Import / Export

- [ ] 컬렉션 · 프로젝트 JSON 내보내기
- [ ] 컬렉션 · 프로젝트 JSON 가져오기
